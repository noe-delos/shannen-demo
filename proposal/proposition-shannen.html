Section 02 — Forfait mensuel

Maintenance opérationnelle.

Trois piliers, des engagements chiffrés, aucun SLA bidon. Tout est livrable et vérifiable.

Pilier A

Veille IA & dépendances

  • Surveillance hebdomadaire de 4 sources : changelog Anthropic, blog ElevenLabs, releases GitHub Next.js, advisories Supabase.
  • SLA migration modèle : nouveau modèle stable en prod sous 14 jours après release ou avant la date de dépréciation, le premier des deux.
  • Trimestriel : audit npm audit + montée Next / Supabase / Radix sur branche mission1_neocell, recette en preview Vercel avant main.
Pilier B

Bug-fix garanti

  • SLA temps de réponse : < 24 h ouvrées non-bloquant, < 4 h ouvrées critique.
  • Définition critique : application inaccessible, perte de données, paiement bloqué.
  • Communication async : email + canal partagé (Slack / Notion). Pas d'astreinte téléphonique.
  • 1 release de patches / mois minimum, déployée en preview avant prod.
Pilier C

Optimisation crédits ElevenLabs

  • Plafonds par utilisateur : ajout colonne users.monthly_credit_limit (défaut 50 000), vérification au démarrage de simulation, message UX dédié si dépassement.
  • Ajustements TTS : stability 0.5 → 0.3, similarity_boost 0.8 → 0.6. Audit chiffré : −10 000 crédits / mois sans baisse de qualité perceptible.
  • Rapport mensuel envoyé le 1er du mois : conso par user, top 5 consommateurs, projection 30 jours, alerte si > 80 % du quota.
Engagement Si la consommation moyenne par utilisateur ne baisse pas d'au moins X % au bout de 6 mois, un mois de maintenance offert.
Exemple — migration ajoutée dès J+7supabase/migrations/limit_credits.sql
ALTER TABLE users ADD COLUMN monthly_credit_limit integer DEFAULT 50000; -- Hook au démarrage de simulation SELECT SUM(duration_seconds) FROM conversations WHERE user_id = $1 AND created_at >= date_trunc('month', NOW()); -- → bloquer si total > monthly_credit_limit

Hors-périmètre — explicitement exclu du forfait

  • Nouvelles fonctionnalitésTraitées en avenant ou bank d'heures.
  • Support utilisateur finalPris en charge par l'équipe Shannen.
  • Astreinte 24/7L'application n'est pas une infrastructure critique.
